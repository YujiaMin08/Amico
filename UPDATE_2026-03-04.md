# Amico Update Notes (2026-03-04)

---

## 中文更新说明

### 1) 桌面端与 Web 端连接能力增强

- `desktop/main.js` 新增了可配置的 Web 连接策略：
  - 支持 `AMICO_WEB_APP_URL` 强制指定 Web 地址（例如 `http://localhost:3010`）。
  - 支持 `AMICO_WEB_APP_PORTS` 以逗号分隔的端口探测列表（默认仍为 `3000,3001,3002`）。
- 在未指定强制 URL 时，仍保留自动探测可用端口并连接本地 Next.js 的逻辑。

### 2) 桌宠窗口显示与交互优化

- 新增 `showPetWindow()` 统一管理桌宠显示行为：
  - 窗口置顶与层级提升更稳定。
  - 在全屏/多工作区场景下显示行为更可控。
- 新增快捷键 `CommandOrControl+Shift+A`：
  - 可快速切换显示/隐藏桌宠窗口。
- 在应用退出前增加全局快捷键注销，避免快捷键残留。

### 3) IPC 通道扩展（Web <-> Electron）

- 新增 `load-pet-with-animations`：
  - 支持从一组动画 URL 中自动挑选可下载资源并加载到桌宠窗口。
  - 自动跳过 `blob:` 地址，降低桌面端下载失败概率。
- 新增 `delete-character-data`、`show-pet`、`hide-pet` IPC 事件。

### 4) Preload 安全桥接能力补充

- `desktop/preload.js` 暴露了新接口：
  - `loadPetWithAnimations`
  - `deleteCharacterData`
  - `showPet`
  - `hidePet`
- 与主进程新增 IPC 逻辑保持一致，供前端页面直接调用。

### 5) 桌宠渲染表现微调

- `desktop/pet-window/index.html` 增加 `importmap`，将 `three` 与 `three/addons` 作为模块别名使用。
- `desktop/pet-window/renderer.js` 调整：
  - 相机距离与高度；
  - 模型缩放基准；
  - 模型落地偏移；
- 目标是让角色默认构图和体感尺寸更自然。

### 6) Web 端确认加载流程增强

- `src/app/page.tsx` 中，在 Electron 环境下确认加载角色时：
  - 若动画 URL 全为 `blob:`，会自动回退到 gallery 中可持久访问的远程 URL（如 `lastModelUrl`）。
  - 提升「网页生成 -> 桌面加载」的成功率。

### 7) Electron 启动与依赖更新

- `desktop/package.json`：
  - `start/dev` 脚本改为通过 `node node_modules/electron/cli.js` 启动。
  - `electron` 版本更新到 `^33.4.11`。
- `desktop/package-lock.json` 已同步依赖锁定信息。

### 8) 资源补充

- 新增 `public/Amico.mp4`（演示视频资源）。

### 9) 影响文件清单

- `desktop/main.js`
- `desktop/package.json`
- `desktop/package-lock.json`
- `desktop/preload.js`
- `desktop/pet-window/index.html`

- `public/Amico.mp4`

## English Update Notes

### 1) Stronger Desktop-to-Web Connection Strategy

- `desktop/main.js` now supports configurable web target resolution:
  - `AMICO_WEB_APP_URL` to force a specific web endpoint (for example, `http://localhost:3010`).
  - `AMICO_WEB_APP_PORTS` as a comma-separated probe list (default remains `3000,3001,3002`).
- If no forced URL is provided, desktop still auto-detects an available local Next.js port.

### 2) Improved Pet Window Visibility and UX

- Added `showPetWindow()` as a shared path for pet window presentation:
  - More stable always-on-top behavior.
  - Better handling across fullscreen and multiple workspaces.
- Added global shortcut `CommandOrControl+Shift+A` to toggle pet visibility quickly.
- Added global shortcut cleanup before app quit.

### 3) IPC Expansion (Web <-> Electron)

- Added `load-pet-with-animations`:
  - Picks a downloadable animation/model URL from provided candidates.
  - Skips `blob:` URLs automatically to reduce desktop loading failures.
- Added `delete-character-data`, `show-pet`, and `hide-pet` channels.

### 4) Preload Bridge Enhancements

- `desktop/preload.js` now exposes:
  - `loadPetWithAnimations`
  - `deleteCharacterData`
  - `showPet`
  - `hidePet`
- These methods align with new main-process IPC handlers for direct web usage.

### 5) Pet Renderer Adjustments

- `desktop/pet-window/index.html` now includes an `importmap` for `three` and `three/addons`.
- `desktop/pet-window/renderer.js` tuning updates:
  - Camera distance/height
  - Model scaling baseline
  - Ground offset
- Goal: better default framing and more natural on-screen pet size.

### 6) Web Confirm Flow Reliability

- In `src/app/page.tsx`, when confirming load in Electron:
  - If all animation URLs are `blob:` URLs, logic now falls back to persistent gallery URLs (e.g., `lastModelUrl`) when available.
  - This improves the reliability of the "web generation -> desktop load" path.

### 7) Electron Startup and Dependency Update

- `desktop/package.json` updates:
  - `start/dev` now use `node node_modules/electron/cli.js`.
  - `electron` bumped to `^33.4.11`.
- `desktop/package-lock.json` updated accordingly.

### 8) Asset Addition

- Added `public/Amico.mp4` as a demo video asset.

### 9) Files Updated

- `desktop/main.js`
- `desktop/package.json`
- `desktop/package-lock.json`
- `desktop/preload.js`
- `desktop/pet-window/index.html`

- `public/Amico.mp4`
